# قواعد الالتزام بنمط المشروع والتصميم (Project Code Style & Alignment Rules)

- **دراسة الكود الحالي أولاً**: قبل كتابة أي كود جديد أو تعديل كود قائم (سواء كان Controller أو Blade أو CSS)، يجب عليك أولاً البحث في الملفات المشابهة داخل المشروع وتقليد نفس البناء والأسلوب تماماً وعدم كتابة حلول مختلفة.
- **التوافق مع اتجاه اللغة العربية (RTL)**: لا تفرض فئات محاذاة نصوص مثل `text-right` أو `text-left` من تلقاء نفسك. تحقق دائماً من كيفية معالجة القوالب الافتراضية للاتجاه في المشروع واتبعها لتجنب إفساد التنسيق.
- **مطابقة رسائل التنبيهات والأخطاء**: عند التعامل مع التحقق من البيانات (Validation) أو الرسائل، استخدم دائماً نفس الكلاسات والهياكل الموجودة بالفعل في المشروع (مثل استخدام `@include('admin.errors.errors')`).
- **عدم الاجتهاد الشخصي في التصميم**: التزم بالهيكل العام والألوان والفئات (Classes) المستخدمة في الصفحات المجاورة، واجعل أي عنصر جديد يبدو وكأنه جزء أصيل من المشروع منذ البداية.
